### Key Findings
- **[CNTRLPLANE-3167](https://redhat.atlassian.net/browse/CNTRLPLANE-3167)**: OADP plugin key remapping clarified — source Secret always `cloud-credentials` with key `cloud`; plugin copies to HO NS remapping `cloud`→`credentials` AND preserving `cloud`. Controller reads `credentials` for S3 STS and `cloud` for Azure WI. Also: IAM actions `s3:CreateMultipartUpload` etc. are NOT valid IAM action names — `s3:PutObject` authorizes all multipart operations.
- **[CNTRLPLANE-3167](https://redhat.atlassian.net/browse/CNTRLPLANE-3167)** (OADP plugin): Created [oadp-plugin#247](https://github.com/openshift/hypershift-oadp-plugin/pull/247) — `ParseAWSCredentialData` auto-detects 3 credential formats (bare ARN, INI with `role_arn`, static keys); standalone Velero detection via DPA CRD check; credential fallback when BSL has no `.spec.credential`.
- **[#8246](https://github.com/openshift/hypershift/pull/8246)** (OCPBUGS-81686): v2 auth validation — **blocking**: `context.Context` must be first parameter in `GenerateAuthConfig`, and missing nil guard on exported function will panic on `spec.OIDCProviders`. Validation logic duplicated ~25 lines between v2/kas and support/validations.
- **[#6961](https://github.com/openshift/hypershift/pull/6961)** (OCPBUGS-61291): TestHAEtcdChaos was giving false positives — sub-test functions return `func(*testing.T)` but were called without wiring the return value into `t.Run`, so validations never executed.
- **[#8006](https://github.com/openshift/hypershift/pull/8006)**: grpc security fix — authorization bypass where malformed `:path` headers (missing leading slash) could bypass path-based deny rules in `grpc/authz` interceptors.
