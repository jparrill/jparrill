### Key Findings
- **[#8305](https://github.com/openshift/hypershift/pull/8305)** (CNTRLPLANE-2539): Moving CAPI provider Role into `ReconcileCredentials()` adds unnecessary GET on every reconcile; TOCTOU race when two HCs share `agentNamespace` and delete concurrently — both can delete the shared Role while a provider still needs it.
- **[#8399](https://github.com/openshift/hypershift/pull/8399)**: `--nogpgcheck` in Containerfile disables RPM signature verification — must import GPG key instead. Installroot pulls ~59 transitive packages bloating rpms.lock.yaml.
- **[#8321](https://github.com/openshift/hypershift/pull/8321)** (OCPBUGS-60093): Pre-existing `maps.Clone` bug in `preserveOriginalMetadata` — shared map pointer between existing and desired objects causes `DeepDerivative` to miss label value changes. Present on main since March 2025.
- **[#8296](https://github.com/openshift/hypershift/pull/8296)** (OCPBUGS-63720): Semantic divergence with AWS orphan logic — Azure binary `ValidCredentials()` treats unknown (no conditions) as valid, so machines won't be orphaned during early reconciliation.
- **[#7590](https://github.com/openshift/hypershift/pull/7590)** (CNTRLPLANE-2207): CAPI 1.11 upgrade — confirmed annotation nil-check inversion fix for real nil-map panic; dead code `GetBackwardCompatibleCAPIImage()` should be deleted; silent fail-open on JSON unmarshal in conversion-data path.
- **[#7749](https://github.com/openshift/hypershift/pull/7749)** (CNTRLPLANE-2775): `ValidateCPOMetrics` in E2E `after()` hook runs on ~21 tests — any flake fails all. Exported mutable `KASRequestDurationBuckets` can corrupt histogram buckets.
- **[#8204](https://github.com/openshift/hypershift/pull/8204)**: PLS predicate semantic change in v2 migration — uses only `groupPredicates` with nil per-item predicates, passes if *any* PLS has the field vs v1 requiring *all*. Unnecessary `Ordered` decorator prevents parallel execution.
